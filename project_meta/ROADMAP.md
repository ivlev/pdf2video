# Road Map проекта pdf2video

Проект для создания динамичных видео-презентаций из статических PDF-файлов.

## 🟢 Этап 1: Базовый функционал (MVP) — ВЫПОЛНЕНО
- [x] Извлечение страниц PDF в изображения (используя `go-fitz`).
- [x] Генерация видео-сегментов через FFmpeg.
- [x] Эффект Ken Burns (плавное приближение/зум).
- [x] Склейка сегментов в итоговый MP4.
- [x] Настройка параметров через флаги командной строки (`-input`, `-output`, `-duration`, `-fps`).

## 🟢 Этап 2: Улучшение визуализации и производительности — ВЫПОЛНЕНО
- [x] **Параллелизация**: Рендеринг страниц в несколько потоков (Go routines).
- [x] **Пропорции (Aspect Ratio)**: Автоматическое сохранение геометрических пропорций PDF (Letterbox/Pillarbox) в выходном видео.
- [x] **Эффекты переходов**: Добавление crossfade, wipe, slide и других эффектов xfade.
- [x] **Рефакторинг**: Переход на ООП-архитектуру (Интерфейсы, Оркестратор).
- [x] **Улучшенный зум (Zoom Drift)**: Zoom In/Out, плавное панорамирование, настройка интенсивности.
- [x] **Оптимизация FFmpeg**: Поддержка аппаратного ускорения (NVENC/VideoToolbox).
- [x] **Debug Overlay**: Режим отладки для визуализации траектории камеры и статистики.
- [x] **Оптимизация PDF**: Переиспользование дескрипторов PDF через `sync.Pool` (устранение I/O бутылочного горлышка).

## 🟢 Этап 3: Аудио и контент — ВЫПОЛНЕНО
- [x] **Поддержка аудио**: Наложение фоновой музыки с автоматической синхронизацией длительности.
- [x] **Рандомизация длительности**: Разная длина клипов (+/- 15%) для динамичности.
- [x] **Поддержка изображений**: Возможность сборки видео из папки с картинками (JPG/PNG).
- [x] **Smart Zoom**: Автоматический анализ изображений и генерация YAML сценариев.
- [x] **Background Audio**: Наложение фоновой музыки с автоматическим зацикливанием и фейдами.
- [x] **Graceful Shutdown**: Корректная остановка FFmpeg и очистка ресурсов по Ctrl+C (Context-aware).
- [ ] Voice-over: Синхронизация с аудио-комментариями (TTS).

## 🟡 Этап 4: Экосистема и интерфейс — В ПРОЦЕССЕ
- [x] **Пресеты форматов**: 16:9 (YouTube), 9:16 (Shorts/TikTok), 4:5 (Instagram).
- [x] **Scenario Config**: Полная поддержка YAML сценариев для управления камерой.
- [x] **Статистика**: Логирование прогресса и итоговый отчет о генерации.
- [x] **Robustness**: Использование `-filter_script` и оптимизация выражений $O(N)$ для сложных сцен.
- [x] **Black Screens**: Сборка вступительного и финального черных экранов (Intro/Outro).
- [ ] **Типографика и Титры**: Наложение анимированного текста на черные экраны через встроенные субтитры ASS.
- [ ] **GUI/Web**: Простой графический интерфейс для управления процессом.
- [ ] **Водяные знаки**: Наложение логотипа или текста поверх видео.

## 🔴 Этап 5: Оптимизация и Стабильность (В ПЛАНЕ) 
- [x] **Adaptive DPI (P0)**: Динамический расчет DPI под целевое разрешение видео.
- [x] **Buffer Pooling (P1)**: Переиспользование памяти через `sync.Pool` для уменьшения нагрузки на GC.
- [x] **Persistent Cache (P2)**: Кэширование отрендеренных страниц PDF на диске.
- [x] **Memory Manager (P3)**: Бюджетирование RAM и контроль лимитов для предотвращения OOM.
- [ ] **Tiled Rendering**: Рендеринг сверхтяжелых PDF страниц по частям (тайлам).

## 🔵 Этап 6: Quality Assurance & CI/CD (В ПЛАНЕ)
- [ ] **System Testing**: Покрытие тестами пакета `internal/system` (Memory Manager, Cache).
- [ ] **FFmpeg Mocking**: Разработка системы тестов для `internal/video` без запуска реального FFmpeg.
- [ ] **Race Detection**: Регулярные тесты на состояние гонок в многопоточных очередях.
- [ ] **CI Pipeline**: Настройка GitHub Actions для автоматической проверки билдов и тестов.

## 🚀 Этап 7: Smart Directing 2.0 — В ПРОЦЕССЕ
*Порядок внедрения определен техническими зависимостями (от данных к визуализации):*

1.  [x] **Enhanced Detector**: Глубокий анализ контента (Edge Density, Color Variance).
2.  [x] **Semantic Scorer**: Оценка информативности блоков.
3.  [x] **Trajectory Optimizer**: Умная сортировка порядка посещения блоков (баланс важности и расстояния).
4.  [ ] **Adaptive Dwell Time**: Динамическое время задержки на основе значимости блока.
5.  [ ] **Path Smoothing**: Сглаживание траекторий полета камеры (Splines/Inertia).

## 🚀 Будущие возможности
- [ ] **Альтернативные детекторы**: OCR (Tesseract) и Saliency (тепловые карты внимания).
- [ ] **Advanced Camera Flights**: Сплайновая интерполяция (Catmull-Rom), адаптивный тайминг по OCR и физика инерции.
- [ ] **TTS-интеграция**: Автоматическая озвучка текста из PDF.
- [ ] **CI/CD Pipeline**: Автоматические бенчмарки и контроль качества при каждом комите.
- [ ] **Plugin System**: Возможность добавления кастомных эффектов через плагины.

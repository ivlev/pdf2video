# PDF2Video: Cinematic Video Generator (v0.91) Mac OS/Linux

Превратите статические PDF и изображения в динамичный видеоряд.

pdf2video — это высокопроизводительная CLI-утилита с аппаратным ускорением, которая не просто склеивает слайды, а выступает в роли виртуального оператора. Она анализирует контент, строит траекторию движения камеры (Smart Zoom) и синхронизирует видео с аудиодорожкой с точностью до кадра.

## ✨ Ключевые особенности

- **Рендеринг по сценарию (YAML):** Полный контроль над камерой (zoom, pan) и таймингом через файлы сценариев.
- **Умная генерация сценариев:** Автоматический анализ страниц PDF (Contrast Detector) для создания динамичных движений камеры.
- **Адаптивный тайминг:** Автоматическое масштабирование длительности сценария под длину выбранного аудиофайла.
- **Кинематографичный Зум (Scenario):** Плавные переходы между ключевыми точками интереса на слайде.
- **Поддержка PDF и Изображений:** Используйте как PDF-файлы, так и папки с изображениями (`.jpg`, `.jpeg`, `.png`) в качестве источника.
- **Аппаратное ускорение:** Автоматическое обнаружение и использование VideoToolbox (Mac) или NVENC (NVIDIA) для сверхбыстрого рендеринга.
- **Graceful Shutdown:** Корректная остановка кодирования по Ctrl+C с очисткой ресурсов.
- **Бесшовные переходы:** Поддержка эффектов `xfade` (fade, wipe, slide, pixelize и др.).
- **Музыкальное сопровождение:** Автоматическая синхронизация длительности видео с вашим аудио-треком.
- **Фоновое аудио (Layering):** Наложение фонового трека (ambient/music) с плавным нарастанием и затуханием громкости.
- **Производительность:** Параллельный рендеринг кадров с использованием пула PDF-документов, **Render Caching**, **Buffer Pooling** и **Memory Budgeting** (защита от OOM).
- **Гибридная архитектура:** Кэширование на уровне страниц (диск) в сочетании с потоковой передачей кадров (RAM Pipes) для исключения лишнего I/O.
- **Масштабируемость:** Оптимизация выражений фильтров (O(N)) и использование `-filter_script` для обхода системных лимитов длины команды.
- **Debug Overlay:** Режим визуализации траектории камеры и технической статистики (отрисовывается на стороне Go для 100% надежности).

## 🛠 Требования

Для работы приложения необходимы:
1. **Go** (версия 1.23 или выше)
2. **FFmpeg** (должен быть доступен в системном PATH)

## 🚀 Быстрый старт

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/ivlev/pdf2video.git
   cd pdf2video
   ```

2. Подготовьте входные данные:
   - Положите PDF в `input/pdf/` **ИЛИ** изображения в `input/images/`.
   - Положите фоновую музыку в `input/audio/`.

3. Запустите конвертацию:
   ```bash
   go run cmd/pdf2video/main.go
   ```

## 🎬 Рендеринг по сценарию (Advanced)

Этот режим позволяет точно управлять камерой и таймингом. Сценарий — это YAML-файл со списком ключевых кадров для каждого слайда.

1. **Генерация базового сценария:**
   Программа проанализирует PDF и создаст файл в `internal/scenarios/` с детекцией областей интереса.
   ```bash
   go run cmd/pdf2video/main.go -generate-scenario
   ```

2. **Рендеринг по сценарию:**
   Видео будет создано в строгом соответствии с ключевыми кадрами. Если указано аудио, длительность сцен в сценарии автоматически масштабируется.
   ```bash
   go run cmd/pdf2video/main.go -scenario internal/scenarios/scenario_...yaml
   ```

3. **Визуальная отладка:**
   Проверьте точность движения камеры перед финальной сборкой.
   ```bash
   go run cmd/pdf2video/main.go -scenario internal/scenarios/scenario_...yaml -debug
   ```

## 📂 Логика именования файлов

Если флаг `-output` не указан, программа формирует имя в папке `output/`:
- **Если вход — PDF:** `ИМЯ_PDF_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.mp4`
- **Если вход — изображения + аудио:** `ИМЯ_АУДИО_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.mp4`
- **Если вход — изображения без аудио:** `ИМЯ_СВЕЖЕГО_ИЗОБРАЖЕНИЯ_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.mp4`

## ⚙️ Настройка (Флаги)

| Флаг | Описание | По умолчанию |
|------|----------|--------------|
| `-input` | Путь к PDF или папке с изображениями | свежий файл в `input/pdf/` |
| `-preset` | Формат видео (`16:9`, `9:16`, `4:5`) | - |
| `-duration` | Общая длительность видео (сек) | 0 (авто по аудио) |
| `-page-duration` | **Средняя** длительность слайда (вариация ±15%) | 0.3 |
| `-audio` | Путь к аудио-файлу | последняя в `input/audio/` |
| `-black-screen-duration`| Длительность черного экрана в начале и в конце (сек) | `2.0` |
| `-black-screen-transition`| Переход для черного экрана (none для отключения)| как в `-transition`|
| `-zoom-mode` | Тип движения для DefaultEffect | `center` |
| `-zoom-speed` | Скорость зума для DefaultEffect | `0.001` |
| `-transition` | Тип перехода (`fade`, `wipeleft`, `slideup`, `pixelize`) | `fade` |
| `-fade` | Длительность эффекта перехода (сек) | `0.5` |
| `-dpi` | Качество рендеринга PDF | `300` |
| `-quality` | Качество (x264: CRF 1-51, VideoToolbox: битрейт=Q*100кбит/с) | `авто` |
| `-analyze-mode` | Режим анализа (`contrast`, `ocr`) | `contrast` |
| `-generate-scenario` | Создать YAML-сценарий на основе анализа PDF | `false` |
| `-scenario` | Использовать YAML-сценарий для рендеринга | `авто` |
| `-stats` | Вывод метрик производительности | `false` |
| `-debug` | Режим отладки: отрисовка траектории камеры и статистики | `false` |
| `-outro-duration` | Длительность возврата к 1:1 перед переходом (сек) | `1.0` |
| `-bg-audio` | Путь к фоновому аудио | свежий в `input/background/` |
| `-bg-volume` | Громкость фонового аудио (0.0 - 1.0) | `0.3` |
| `-workers` | Количество потоков обработки | runtime.NumCPU |

> **Примечание:** Длительность каждого слайда автоматически варьируется в пределах ±15% от `-page-duration` для создания эффекта "живого" монтажа.

## 🏗 Архитектура

Проект построен на модульной объектно-ориентированной архитектуре:
- `internal/source`: Унифицированный интерфейс для PDF и изображений.
- `internal/video`: Взаимодействие с FFmpeg (кодирование).
- `internal/effects`: Генерация визуальных фильтров и анимаций.
- `internal/engine`: Оркестратор процесса создания видео.
- `internal/system`: Системные утилиты и работа с файлами.
